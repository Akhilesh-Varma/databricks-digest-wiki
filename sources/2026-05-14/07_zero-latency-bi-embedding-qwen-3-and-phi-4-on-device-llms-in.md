## Summary
The article explores embedding on-device large language models, specifically Qwen-3 and Phi-4, directly into business intelligence platforms Power BI and Qlik Cloud to achieve zero-latency AI-assisted analytics. The traditional approach routes user queries through remote API endpoints, introducing latency and privacy concerns, whereas on-device LLMs process queries locally within the BI tool itself. The piece outlines the technical architecture and integration patterns required to run these compact models inside Power BI and Qlik Cloud environments. This approach aims to eliminate network round-trips and keep sensitive data on-premises or on-device.

## Key claims
- Traditional AI-assisted BI routes user questions to remote API endpoints, introducing latency.
- Embedding on-device LLMs like Qwen-3 and Phi-4 into BI tools can achieve zero-latency query responses.
- Qwen-3 and Phi-4 are compact LLMs suitable for on-device deployment within BI platforms.
- Power BI and Qlik Cloud are the two BI platforms targeted for on-device LLM integration.
- On-device LLM deployment addresses privacy concerns by keeping data local rather than sending it to external APIs.
