## Summary
Databricks has announced the Public Preview of real-time mode in Lakeflow Spark Declarative Pipelines. This feature enables end-to-end latency as low as five milliseconds, targeting operational streaming workloads like fraud detection, real-time personalization, and instant decision-making. Real-time mode is powered by the new update_flow decorator, which writes to sinks using update output mode. Notably, it supports stateful aggregations without requiring a watermark, simplifying streaming pipeline development.

## Key claims
- Real-time mode in Lakeflow Spark Declarative Pipelines is now in Public Preview.
- Real-time mode achieves end-to-end latency as low as five milliseconds.
- Target use cases include fraud detection, real-time personalization, and instant decision-making.
- Real-time mode uses the new update_flow decorator to write to sinks using update output mode.
- The update_flow API supports stateful aggregations without requiring a watermark.
