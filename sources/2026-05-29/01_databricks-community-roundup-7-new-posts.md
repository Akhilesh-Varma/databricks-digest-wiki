## Summary
This Databricks community roundup covers seven new technical blog posts spanning AI agent management, query performance, data experimentation, pipeline editing, ETL frameworks, platform evolution, and MLOps. Key topics include using Unity AI Gateway to manage agent and tool sprawl via the Model Context Protocol, performance tuning tips for Databricks Serverless SQL, and the general availability of the Lakeflow Pipelines Editor. The posts collectively reflect Databricks' expansion beyond its big data roots into AI, machine learning operations, and enterprise data workflows. A free MLOps course from Databricks Academy is also highlighted for ML practitioners.

## Key claims
- Unity AI Gateway in Databricks can be used to manage agent and tool sprawl in enterprise AI deployments.
- Model Context Protocol (MCP) enables connecting multiple tools and data sources to build AI agents such as a travel-planning agent.
- Databricks Serverless SQL (DBSQL) is positioned as Databricks' latest offering for building data warehouses on the Lakehouse.
- The Lakeflow Pipelines Editor is now generally available.
- Databricks is described as having evolved beyond being just a big data platform.
- Databricks Academy offers a free Machine Learning Operations course covering model management across the ML lifecycle.
- A metadata-driven ETL framework approach is presented for modern data-driven enterprises using Databricks.
