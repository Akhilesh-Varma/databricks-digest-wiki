## Summary
Databricks announced that SQL alerts will be available by default for workspaces that have the compliance security profile enabled, starting June 2026. SQL alerts allow users to monitor data and key performance indicators by running scheduled queries, evaluating conditions, and sending notifications when those conditions are met. Common use cases include detecting KPI drifts, identifying anomalies, and surfacing data quality issues. This change expands access to SQL alerting capabilities within security-hardened workspace configurations.

## Key claims
- Databricks SQL alerts will be enabled by default for workspaces with the compliance security profile starting June 2026.
- SQL alerts work by running queries on a schedule and evaluating user-defined conditions.
- Notifications are sent to recipients when alert conditions are met.
- Common use cases for SQL alerts include monitoring KPI drifts, detecting anomalies, and identifying data quality issues.
- Previously, SQL alerts were not available by default in workspaces with the compliance security profile enabled.
