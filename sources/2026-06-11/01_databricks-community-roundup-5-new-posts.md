## Summary
This Databricks Community roundup highlights five new posts covering a range of technical topics. The posts include query performance tuning tips for Databricks Serverless SQL, best practices for safe data experimentation, and a metadata-driven ETL framework guide. Additional posts address a compute issue when refreshing Power BI datasets from Databricks workflows, and a free Databricks Academy course on AI BI for data analysts. Together, the posts reflect common use cases and learning resources within the Databricks ecosystem.

## Key claims
- Databricks Serverless SQL (DBSQL) is Databricks' latest offering for building data warehouses on the Lakehouse.
- Databricks Academy offers a free AI BI for Data Analysts course covering AI BI Dashboards and AI BI Genie.
- Calling a Power BI refresh from a Databricks workflow job causes the job to wait until the refresh is complete, unlike Azure Data Factory's REST API approach.
- A metadata-driven ETL framework in Databricks is presented as a solution for managing complex data flows in modern enterprises.
- Best practices for safe data experimentation in Databricks are aimed at developers and data scientists building models and data pipelines.
