## Summary
Databricks has announced native support for OpenAI's GPT 5.5 and Codex models, enabling users to leverage these advanced AI capabilities for coding, agent development, and document intelligence. The integration is managed through Databricks' Unity AI Gateway, providing a unified control plane for governance features like permissions, rate limits, and guardrails across AWS, Azure, and GCP. Key features include direct use of GPT 5.5 for coding with Codex, natural language analytics via Genie powered by Unity Catalog semantics, building sophisticated agents with Agent Bricks, and integrating GenAI into Spark Declarative Pipelines for ETL and document processing.

## Key claims
- Databricks now natively supports OpenAI's GPT 5.5 and Codex models.
- Users can leverage GPT 5.5 for coding workflows with Codex, enterprise agents, and document pipelines.
- The Unity AI Gateway provides unified governance for GPT 5.5 and Codex, including permissions, rate limits, and guardrails.
- Genie, powered by GPT 5.5 and Unity Catalog semantics, enables natural language analytics on governed enterprise data.
- Agent Bricks allows the creation of GPT 5.5-powered agents for multi-step workflows and deployment as Databricks Apps.
- GPT 5.5 can be integrated into Spark Declarative Pipelines for GenAI ETL and document intelligence with built-in governance.
