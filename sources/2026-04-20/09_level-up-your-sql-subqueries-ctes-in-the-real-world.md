## Summary
This article discusses the use of subqueries and Common Table Expressions (CTEs) in SQL to simplify complex queries. It explains how subqueries can be used to compute single values, filter results, and refactor queries. The article also covers different types of subqueries, including scalar, single row, multi-row, and correlated subqueries. The goal is to help readers understand how to use subqueries and CTEs to improve their SQL skills. The article is practical and Postgres-friendly, with examples to help readers mentally picture the rows being produced at each step. By the end of the article, readers should be able to refactor queries and use subqueries and CTEs effectively.

## Key claims
- Subqueries can be used to compute single values, filter results, and refactor queries.
- Common Table Expressions (CTEs) can be used to simplify complex queries.
- Scalar subqueries return exactly one row and one column, and can be used like a single value.
- Single row subqueries return one row, which can be compared as a unit.
- Multi-row subqueries return many rows, and can be used to check if a value is in a set.
- Correlated subqueries are used to compute a single value per row.
