## Summary
The article describes building a unified Retrieval-Augmented Generation (RAG) pipeline using three Databricks SQL AI functions: ai_parsed_document v2.0, ai_extract v2.1, and ai_prep_search. These functions are combined into a single SQL-native pipeline designed for hybrid document search. The approach leverages Databricks SQL to handle document parsing, structured data extraction, and search preparation without leaving the SQL environment. The article was published on Medium as part of the Databricks publication.

## Key claims
- A unified RAG pipeline can be built using three Databricks SQL AI functions: ai_parsed_document v2.0, ai_extract v2.1, and ai_prep_search.
- The pipeline is entirely SQL-native, requiring no external orchestration outside of Databricks SQL.
- ai_parsed_document v2.0 handles document parsing within the pipeline.
- ai_extract v2.1 is used for structured data extraction from parsed documents.
- ai_prep_search prepares data for hybrid document search.
- The combined pipeline supports hybrid document search use cases.
