## Summary
The article describes a staff engineer's journey from traditional, brittle ETL pipelines to an 'agentic data serving' architecture designed to handle the unpredictable, heavily nested JSON data generated by AI and LLM features. The author argues that the Modern Data Stack's decoupled approach is too fragile for AI-native workloads, where schema changes cause cascading failures. The proposed solution pairs S3 storage with DuckDB's native JSON and schema-agnostic Parquet handling to eliminate failure-prone parsing steps. The Model Context Protocol (MCP) replaces custom LangChain tooling by providing a standard interface for AI agents to discover schemas and execute queries securely. The result is a minimal-infrastructure, consumption-based agentic data platform where AI agents can autonomously query data.

## Key claims
- Data teams spend up to 80% of their time moving and cleaning data rather than performing analysis, according to a Forbes survey.
- The ETL market is projected to reach $20.1 billion by 2032 at a 13% CAGR.
- Traditional decoupled ETL pipelines in the Modern Data Stack are too brittle to handle the unpredictable, heavily nested JSON data produced by AI and LLM features.
- DuckDB's native JSON handling and schema-agnostic Parquet reading eliminate failure-prone parsing steps when paired with S3 storage.
- The open Model Context Protocol (MCP) provides a standard interface for AI agents to discover schemas and execute queries, replacing custom LangChain tools.
- Agentic data serving focuses on dynamic query routing and semantic discovery, enabling AI agents to autonomously discover and query data.
- DuckDB's embeddable architecture makes it practical to connect agents directly to data with minimal infrastructure overhead and elastic, consumption-based compute.
