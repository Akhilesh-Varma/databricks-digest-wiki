## Summary
Databricks has announced that the Auto Time-to-Live (auto-TTL) feature is now generally available. Auto-TTL automatically deletes rows from tables managed in Unity Catalog based on a DATE or TIMESTAMP column. The feature supports Delta Lake managed tables, Apache Iceberg managed tables, and streaming tables. This capability simplifies data lifecycle management by enabling automatic row expiration without manual intervention.

## Key claims
- Auto time-to-live (auto-TTL) is now generally available on Databricks.
- Auto-TTL automatically deletes rows from Unity Catalog managed tables based on a DATE or TIMESTAMP column.
- Auto-TTL supports Delta Lake managed tables, Apache Iceberg managed tables, and streaming tables.
- The feature is scoped to Unity Catalog managed tables only.
