## Summary
Charlie Labs has pivoted from building coding agents to creating Daemons, a new product category designed to clean up after agents and reduce operational drag. The company developed Charlie, a coding agent, but found that agents create more work, leading to outdated code, documentation drift, and stale dependencies. Daemons are added to a codebase by adding an .md file to the repo and run in a set-it-and-forget-it way. The goal is to make developers' lives easier and accelerate projects. Daemons are designed for teams using agents like Claude, Codex, Cursor, or Cline.
