## Summary
This article is a community roundup from Databricks featuring seven new technical blog posts. Topics covered include query performance tuning for Databricks Serverless SQL, best practices for safe data experimentation, and using large language models for customer service analytics. Additional posts address a metadata-driven ETL framework, automatic Git deployments for Databricks Apps, and the Lakeflow Connect feature for MySQL ingestion. The roundup reflects a broad range of practical guidance for Databricks platform users.

## Key claims
- Databricks Serverless SQL (DBSQL) is positioned as Databricks' latest offering for building data warehouses on the Lakehouse.
- Databricks recommends limiting each Lakeflow Connect ingestion pipeline for MySQL to around 250 tables, with validated testing up to 1 TB of snapshot data.
- Lakeflow Connect for MySQL is currently in public preview.
- Automatic Git deployments for Databricks Apps can be triggered from GitHub, streamlining the deploy workflow.
- Large language models (LLMs) are being applied to customer service analytics involving unstructured data such as call transcripts and chat messages.
- A metadata-driven ETL framework is presented as a modern approach for managing complex data flows in data-driven enterprises.
