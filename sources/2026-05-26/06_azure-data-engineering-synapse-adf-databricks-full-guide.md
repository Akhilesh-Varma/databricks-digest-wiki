## Summary
This article is a comprehensive guide to Azure data engineering covering three core services: Azure Data Factory for orchestration and pipeline movement, Azure Synapse Analytics as a unified workspace combining Dedicated SQL Pools, Serverless SQL, and Spark Pools, and Azure Databricks as a lakehouse compute engine. It explains when to use each service and how they integrate with ADLS Gen2 as a shared storage layer. The guide also covers Delta Lake medallion architecture, Unity Catalog governance, and the broader Azure data platform stack including Event Hubs, Stream Analytics, Purview, Power BI, and Microsoft Fabric. Each section includes worked examples and interview-style problems with code, traces, and explanations.

## Key claims
- Azure data engineering in 2026 is anchored by three flagship services: Azure Data Factory, Azure Synapse Analytics, and Azure Databricks.
- Azure Synapse Analytics fuses Dedicated SQL Pools, Serverless SQL, and Spark Pools on a single ADLS Gen2 lake.
- Azure Databricks serves as the lakehouse compute engine supporting clusters, notebooks, Delta Lake, and Unity Catalog.
- ADLS Gen2 with hierarchical namespace is the shared storage layer underlying all three core Azure data engineering services.
- The broader Azure data platform stack includes Event Hubs, Stream Analytics, Purview, Power BI, and Microsoft Fabric.
- Delta Lake medallion architecture is a key pattern covered for production data engineering work on Azure.
- Unity Catalog provides governance capabilities within the Azure Databricks lakehouse environment.
