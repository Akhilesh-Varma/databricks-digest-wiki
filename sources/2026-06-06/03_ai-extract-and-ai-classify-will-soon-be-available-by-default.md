## Summary
Databricks announced that the ai_extract and ai_classify SQL AI Functions will become available by default in mid-July 2026 for workspaces that have the compliance security profile enabled along with specific regulatory controls. The ai_extract function allows users to extract structured fields from unstructured text based on a user-defined schema, while ai_classify enables text classification into user-defined labels for routing, tagging, and document categorization. These features will support a broad range of compliance frameworks including HIPAA, PCI-DSS, FedRAMP Moderate, and others. Users can also leverage Agent Bricks to build information extraction and classification agents.

## Key claims
- ai_extract and ai_classify SQL AI Functions will be available by default for compliant workspaces in mid-July 2026.
- The rollout applies to workspaces with the compliance security profile enabled and specific controls selected, including HIPAA, PCI-DSS, FedRAMP Moderate, CCCS Medium Protected B, UK Cyber Essentials Plus, HITRUST, IRAP, C5, and TISAX.
- ai_extract is used to extract structured fields from unstructured text against a user-defined schema.
- ai_classify is used to classify text into user-defined labels for routing, tagging, and document categorization workflows.
- Agent Bricks can be used to create information extraction agents and classification agents as an alternative approach.
