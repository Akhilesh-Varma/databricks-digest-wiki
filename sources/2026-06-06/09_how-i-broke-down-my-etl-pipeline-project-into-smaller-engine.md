## Summary
The author describes breaking down an ETL pipeline project into smaller, focused engineering exercises to better understand modern data systems. The first exercise covered the Extract phase, emphasizing data ingestion, consistency, and reliability. The second exercise tackled the Transform phase, including cleaning records, handling missing values, schema design, and data normalization. Key takeaways highlight that ETL pipelines are about data trust and scalability, not just data movement. Future work will address the Load phase, pipeline orchestration, and scalability improvements.

## Key claims
- Breaking an ETL project into smaller exercises made engineering concepts easier to learn than tackling the full system at once.
- The Extract phase involves more than reading data; it requires ensuring consistency, structure, and reliability of incoming records.
- The Transform phase was identified as the most complex stage, involving cleaning, null handling, restructuring, and field standardization.
- A poorly designed transformation layer can create downstream problems for analytics, reporting, and other consuming services.
- ETL pipelines are fundamentally about ensuring data trust and building reliable, scalable processing workflows.
- The next planned phase will cover loading transformed data, pipeline orchestration, and scalability improvements.
