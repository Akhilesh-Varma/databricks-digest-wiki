## Summary
This article introduces Linux fundamentals specifically tailored for data engineers. It covers core topics including file system navigation, file and directory management, permissions, process management, networking, shell scripting, and automation. The article argues that Linux is essential because most data platforms—such as Apache Hadoop, Spark, Kafka, and databases like PostgreSQL and MySQL—run on Linux-based systems. Practical examples are provided throughout to illustrate real-world data engineering tasks such as managing pipelines, scheduling jobs, and organizing project directories. The piece positions Linux knowledge as a foundational requirement alongside Python and SQL for aspiring data engineers.

## Key claims
- Linux dominates the server and cloud computing ecosystem used in data engineering.
- Technologies such as Apache Hadoop, Apache Spark, Apache Kafka, PostgreSQL, MySQL, Docker, and Kubernetes are typically deployed on Linux servers.
- Data engineers must understand Linux for tasks including accessing remote servers, monitoring pipelines, scheduling automated jobs, and deploying applications.
- Linux uses a hierarchical directory structure beginning with the root directory, unlike Windows.
- Shell scripting and automation are key Linux skills for data engineers alongside file management and process management.
- Understanding Linux fundamentals is described as a necessity, not merely an advantage, for aspiring data engineers.
