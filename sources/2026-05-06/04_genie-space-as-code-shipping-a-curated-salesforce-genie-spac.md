## Summary
The article discusses a methodology for deploying Databricks Genie Spaces as code using Databricks Asset Bundles, specifically targeting a Salesforce-integrated use case. The author argues that typical Genie Space demos are fragile because they rely on manual UI configuration rather than reproducible, version-controlled definitions. By treating Genie Space configuration as code within a Databricks Asset Bundle, teams can ship curated, repeatable analytics environments. The piece is published on Medium and appears to be a technical walkthrough aimed at data engineers and analytics engineers working with Databricks.

## Key claims
- Most Genie Space demos fail because they rely on manual UI configuration rather than code-driven, reproducible setups.
- Databricks Asset Bundles can be used to define and ship Genie Spaces as code.
- The article demonstrates a curated Salesforce Genie Space delivered via a Databricks Asset Bundle.
- Treating Genie Space configuration as code enables version control and repeatability.
- Hand-curated joins and synonyms in the UI are cited as fragile practices that the code-based approach aims to replace.
