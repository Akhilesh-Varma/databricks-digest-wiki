## Summary
The paper introduces DPC (Dual-Paradigm Consistency), a training-free multi-agent framework for selecting the best SQL query generated by Large Language Models without requiring an execution oracle. DPC addresses the Generation-Selection Gap—where high Pass@K accuracy fails to translate into Pass@1 accuracy—by reformulating selection as a deterministic verification task on visible data rather than probabilistic guessing on hidden data. Three agents (SLICER, TESTER, and SOLVER) collaborate to construct a Minimal Distinguishing Database (MDD) that exposes logical discrepancies between SQL candidates, then cross-reference SQL execution results against a parallel Python/Pandas solution to break self-correction bias. Experiments on the BIRD and Spider benchmarks across multiple LLMs show DPC achieves up to 2.2 percentage points of absolute accuracy improvement over baselines like Self-Consistency.

## Key claims
- LLMs lack the capability to self-evaluate SQL correctness without an execution oracle, creating a Generation-Selection Gap between Pass@K and Pass@1 accuracy.
- Existing training-free selection methods like Self-Consistency and LLM-as-a-Judge suffer from systematic bias (consensus on hallucinations) and symbolic blindness.
- DPC reformulates SQL candidate selection from a probabilistic guessing task on hidden data into a deterministic verification task on visible data.
- DPC uses three agents—SLICER, TESTER, and SOLVER—to construct a Minimal Distinguishing Database (MDD) that exposes logical discrepancies between SQL candidates.
- A SOLVER agent verifies SQL candidates by cross-referencing their execution results against a parallel Python/Pandas solution to break self-correction bias.
- DPC achieves absolute accuracy improvements of up to 2.2 percentage points over strong baselines like Self-Consistency on BIRD and Spider benchmarks.
