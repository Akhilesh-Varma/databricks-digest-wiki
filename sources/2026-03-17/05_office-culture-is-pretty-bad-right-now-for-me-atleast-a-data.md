## Summary
A solo data engineer at an 80-person startup describes growing workplace stress driven by management's aggressive push for AI adoption using Claude across all departments. Rather than reducing workload, the AI mandate has increased expectations and led to developers working late nights to ship features rapidly. The author feels their core data engineering work is being undermined, as a technical founder built a customer-facing dashboard directly on an OLTP database using TypeScript and React, bypassing the author's Databricks-based OLAP stack. Cost constraints on Databricks prevent real-time data solutions, making the OLTP approach appear superior in their context. The author also notes other developers are taking on data engineering tasks using tools like Dagster, further eroding their role.

## Key claims
- Management mandated company-wide use of Claude AI across all departments at an 80-person startup.
- AI adoption increased developer workload and late-night work rather than reducing it.
- A technical founder built a customer-facing dashboard on an OLTP database using TypeScript and React JS, bypassing the OLAP/Databricks stack.
- OLTP databases with proper indexes can outperform OLAP for real-time use cases in this context.
- Real-time data processing in Databricks was deemed too costly and blocked by the finance team.
